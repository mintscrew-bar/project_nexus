# 롤 내전 오토 밸런싱 기능을 설계하면서 고민했던 것들

안녕하세요. 내전 플랫폼 **NEXUS**를 개발하고 있는 하루마룬입니다.

최근 NEXUS에 새로운 팀 구성 방식으로 **오토 밸런싱**(앱에서는 '자동 밸런스' 모드) 기능을 추가했습니다.

기존에는 내전 팀을 나누려면 보통 경매 드래프트나 스네이크 드래프트처럼 팀장이 직접 선수를 고르는 방식을 사용했습니다. 이런 방식은 확실히 재미가 있습니다. 누가 누구를 먼저 고를지, 어느 팀장이 포인트를 어떻게 쓸지 보는 맛도 있고요.

하지만 매번 내전을 열 때마다 팀장을 정하고, 한 명씩 고르고, 기다리고, 다시 역할을 정하는 과정이 조금 무겁게 느껴질 때도 있었습니다.

특히 가볍게 10인 내전을 열고 싶은 상황에서는 이런 생각이 들곤 했습니다.

> "그냥 버튼 한 번 누르면 적당히 팀 나눠주면 안 되나?"

이번 글에서는 NEXUS의 오토 밸런싱 기능을 만들면서 어떤 문제를 고민했고, 어떤 식으로 구현 방향을 잡았는지 정리해보려고 합니다.

---

## 왜 오토 밸런싱이 필요했나

내전에서 팀을 나누는 방식은 생각보다 중요합니다.

팀이 너무 한쪽으로 기울면 게임이 재미없어지고, 반대로 팀을 나누는 데 시간이 너무 오래 걸리면 시작하기 전부터 피로해집니다.

기존 NEXUS에는 경매 드래프트와 스네이크 드래프트 방식이 있었습니다. 경매 드래프트는 팀장이 포인트를 사용해 선수를 입찰하는 방식이고, 스네이크 드래프트는 팀장이 순서대로 선수를 지명하는 방식입니다.

두 방식 모두 장점이 뚜렷합니다.

- 팀장이 직접 판단해서 팀을 만들 수 있다.
- 진행 과정 자체가 콘텐츠가 된다.
- 참가자 입장에서 보는 재미가 있다.

하지만 단점도 있습니다.

- 팀장을 정해야 한다.
- 한 명씩 고르는 시간이 필요하다.
- 가볍게 내전을 열기에는 진행이 무겁다.
- 참가자가 자주 바뀌면 매번 같은 과정을 반복해야 한다.

그래서 오토 밸런싱은 이런 상황을 목표로 만들었습니다.

> "정원이 다 차면, 방장이 버튼 한 번으로 납득 가능한 팀을 만들 수 있게 하자."

여기서 중요한 표현은 "완벽한 팀"이 아니라 **납득 가능한 팀**입니다.

---

## 단순 랜덤으로는 부족했다

가장 쉬운 방법은 참가자들을 그냥 랜덤으로 섞는 것입니다.

```ts
function shuffle<T>(items: T[]): T[] {
  return [...items].sort(() => Math.random() - 0.5);
}

function createRandomTeams(players: Player[]) {
  const shuffled = shuffle(players);

  return [
    shuffled.slice(0, 5),
    shuffled.slice(5, 10),
  ];
}
```

구현은 간단합니다. 하지만 롤 내전에서는 단순 랜덤이 생각보다 위험합니다.

예를 들어 10명 중에 마스터 1명, 다이아 2명, 골드 4명, 실버 3명이 있다고 가정해보겠습니다. 랜덤으로 나누면 마스터와 다이아가 같은 팀으로 몰릴 수도 있습니다.

그렇게 되면 게임 시작 전부터 이런 말이 나옵니다.

> "이거 팀 너무 기운 거 아닌가요?"

내전은 보통 지인끼리, 서버 멤버끼리, 방송 시청자끼리 진행하는 경우가 많습니다. 그래서 팀이 기울어졌다고 느껴지면 결과보다 과정에서 먼저 불만이 생깁니다.

오토 밸런싱은 빠르게 팀을 나누기 위한 기능이지만, 적어도 "이 정도면 해볼 만하다"는 느낌은 줘야 했습니다. 그래서 단순 랜덤은 제외했습니다.

---

## 티어와 LP를 점수로 바꾸기

그다음으로 생각한 방식은 참가자의 티어와 LP를 점수화하는 것이었습니다.

롤에는 아이언, 브론즈, 실버, 골드, 플래티넘, 에메랄드, 다이아몬드, 마스터 같은 티어가 있고, 각 티어 안에서도 랭크와 LP가 있습니다. 이 정보를 그대로 비교하기는 어렵기 때문에, 내부적으로는 하나의 숫자로 변환해 팀 밸런스를 계산했습니다.

블로그용으로 단순화하면 이런 느낌입니다.

```ts
const TIER_SCORE: Record<string, number> = {
  IRON: 0,
  BRONZE: 400,
  SILVER: 800,
  GOLD: 1200,
  PLATINUM: 1600,
  EMERALD: 2000,
  DIAMOND: 2400,
  MASTER: 2800,
  GRANDMASTER: 3200,
  CHALLENGER: 3600,
};

const RANK_SCORE: Record<string, number> = {
  IV: 0,
  III: 100,
  II: 200,
  I: 300,
};

function calculatePlayerScore(player: Player) {
  const tierScore = TIER_SCORE[player.tier] ?? 0;
  const rankScore = RANK_SCORE[player.rank] ?? 0;
  const lpScore = player.lp ?? 0;

  return tierScore + rankScore + lpScore;
}
```

핵심은 참가자의 실력을 비교 가능한 숫자로 바꾸는 것입니다.

물론 이 점수가 실제 실력을 완벽하게 표현하지는 않습니다. 롤 실력은 티어만으로 설명되지 않고, 챔피언 폭이나 포지션 숙련도, 최근 폼 같은 요소도 영향을 줍니다.

하지만 내전 팀을 빠르게 나누는 상황에서는 너무 복잡한 모델보다, 운영자가 이해할 수 있는 기준이 더 중요하다고 봤습니다.

> "티어와 LP를 기준으로 대략적인 실력 차이를 줄인다."

이 정도가 오토 밸런싱의 첫 번째 목표였습니다.

---

## 팀 점수 차이를 줄이는 방식

참가자 점수를 계산했다면, 이제 팀별 총점을 비교할 수 있습니다. 가장 단순한 평가 함수는 팀 점수의 차이를 보는 것입니다.

```ts
function evaluateTeamBalance(teams: Team[]) {
  const teamScores = teams.map((team) =>
    team.players.reduce((sum, player) => sum + player.score, 0)
  );

  const maxScore = Math.max(...teamScores);
  const minScore = Math.min(...teamScores);

  return maxScore - minScore;
}
```

이 함수는 가장 강한 팀과 가장 약한 팀의 점수 차이를 반환합니다. 값이 작을수록 팀 간 실력 차이가 적다는 뜻입니다.

예를 들어 두 팀의 점수가 각각 8700, 8600이라면 차이는 100입니다. 반대로 9500, 7800이라면 차이는 1700입니다.

오토 밸런싱에서는 가능한 한 이 차이가 작은 조합을 찾는 방향으로 접근했습니다.

---

## 그런데 점수만 맞추면 충분할까?

처음에는 점수 차이만 줄이면 괜찮을 거라고 생각했습니다. 하지만 실제 내전에서는 점수만 맞는다고 좋은 팀이 되는 것은 아니었습니다.

가장 큰 이유는 **포지션**입니다.

롤은 탑, 정글, 미드, 원딜, 서포터 역할이 나뉘어 있습니다. 아무리 팀 점수가 비슷해도 한 팀에 정글 유저가 없거나, 원딜 유저가 몰리면 실제 게임은 불편해집니다.

예를 들어 이런 팀이 나올 수 있습니다.

```
A팀: 탑, 탑, 미드, 서포터, 서포터
B팀: 정글, 정글, 원딜, 원딜, 미드
```

점수는 비슷할 수 있습니다. 하지만 실제 게임을 시작하면 역할 선택 단계에서 바로 문제가 생깁니다.

그래서 오토 밸런싱에서는 티어/LP 점수뿐 아니라 참가자의 주 포지션, 부 포지션도 함께 참고하도록 했습니다.

블로그용으로 단순화하면, 평가 함수에 포지션 패널티를 더하는 식입니다.

```ts
const ROLES = ["TOP", "JUNGLE", "MID", "ADC", "SUPPORT"] as const;

function calculateRolePenalty(team: Team) {
  let penalty = 0;

  for (const role of ROLES) {
    const hasMainRole = team.players.some((player) => player.mainRole === role);
    const hasSubRole = team.players.some((player) => player.subRole === role);

    if (!hasMainRole && !hasSubRole) {
      penalty += 300;
    }
  }

  return penalty;
}
```

이 코드는 팀 안에 특정 포지션을 주/부 포지션으로 가진 사람이 아무도 없으면 패널티를 주는 방식입니다.

실제 구현에서는 더 많은 조건을 고려하지만, 핵심은 간단합니다.

> "점수는 비슷한데 포지션 구성이 너무 망가진 팀은 덜 좋은 조합으로 본다."

---

## 밸런스 점수와 포지션 점수를 함께 보기

이제 팀 조합을 평가할 때 두 가지를 같이 볼 수 있습니다.

- 팀 간 실력 점수 차이
- 포지션 구성 패널티

```ts
function evaluateTeams(teams: Team[]) {
  const balanceGap = evaluateTeamBalance(teams);

  const rolePenalty = teams.reduce(
    (sum, team) => sum + calculateRolePenalty(team),
    0
  );

  return balanceGap + rolePenalty;
}
```

이렇게 하면 점수 차이가 적고, 포지션 구성도 덜 망가진 조합이 더 좋은 조합으로 평가됩니다.

물론 이 방식도 완벽하지는 않습니다. 어떤 유저는 주 포지션이 아니어도 충분히 잘하고, 어떤 유저는 티어는 높지만 특정 라인에서는 약할 수 있습니다. 하지만 모든 정보를 완벽히 알 수 없는 상황에서는, 현재 서비스가 가진 데이터를 최대한 활용하는 쪽이 현실적이었습니다.

## 매번 같은 팀이 나오면 재미없다

오토 밸런싱을 만들면서 또 하나 고민한 부분이 있습니다. 가장 좋은 조합 하나만 고르면 어떻게 될까요?

참가자 구성이 비슷하면 매번 비슷한 팀이 나올 가능성이 높습니다. 운영자 입장에서는 밸런스가 좋다고 느낄 수 있지만, 참가자 입장에서는 이렇게 느낄 수 있습니다.

> "또 저 사람이랑 같은 팀이네?"

그래서 오토 밸런싱에는 어느 정도 랜덤성이 필요했습니다. 다만 완전 랜덤으로 가면 다시 팀 밸런스가 무너집니다. 그래서 선택한 방식은 **좋은 후보군을 여러 개 만들고, 그 안에서 랜덤으로 고르는 것**이었습니다.

```ts
function chooseBalancedTeam(teamsCandidates: Team[][]) {
  const sortedCandidates = teamsCandidates
    .map((teams) => ({
      teams,
      score: evaluateTeams(teams),
    }))
    .sort((a, b) => a.score - b.score);

  const topCandidates = sortedCandidates.slice(0, 10);

  return topCandidates[
    Math.floor(Math.random() * topCandidates.length)
  ].teams;
}
```

이 방식은 단순하지만 효과가 있습니다. 가장 나쁜 조합은 걸러내고, 비교적 괜찮은 조합들 중에서 랜덤으로 하나를 선택합니다.

즉, 밸런스를 크게 해치지 않으면서도 매번 조금씩 다른 팀이 나올 수 있습니다.

## 실제 서비스 코드에서는 더 복잡해진다

블로그에 적은 코드는 이해를 돕기 위해 단순화한 형태입니다. 실제 서비스 코드에서는 이 외에도 여러 처리가 들어갑니다.

- 방 정원이 모두 찼는지 확인
- 모든 참가자가 준비 상태인지 확인
- 참가자의 Riot 계정 정보 조회
- 기존 팀 정보 초기화
- 새 팀 생성과 팀원 배정
- 역할 선택 단계로 전환
- 실시간으로 클라이언트에 상태 전파

서비스 코드는 결국 알고리즘만으로 끝나지 않습니다. 팀을 잘 나누는 것도 중요하지만, 그 결과가 DB에 안전하게 저장되어야 하고, 방에 들어와 있는 사용자들에게 즉시 반영되어야 합니다.

그래서 실제 구현에서는 밸런싱 로직과 트랜잭션 처리, WebSocket 이벤트 처리까지 함께 고려해야 했습니다.

## 완벽한 팀보다 납득 가능한 팀

오토 밸런싱을 만들면서 가장 많이 생각한 질문은 이것이었습니다.

> "완벽한 팀을 만들 수 있을까?"

결론부터 말하면, 어렵다고 생각합니다.

롤 내전에서 완벽한 팀 밸런스를 만드는 것은 생각보다 복잡합니다. 티어와 LP만으로는 실제 실력을 완전히 알 수 없고, 포지션 숙련도도 사람마다 다릅니다. 듀오 경험, 챔피언 폭, 당일 컨디션, 보이스 소통 여부 같은 요소도 영향을 줍니다.

그래서 목표를 조금 바꿨습니다.

> "완벽한 팀은 아니더라도, 사람들이 봤을 때 납득할 수 있는 팀을 만들자."

이 기준이 훨씬 현실적이었습니다. 방장이 버튼을 눌렀을 때 "이 정도면 해볼 만하네"라는 느낌이 들면, 오토 밸런싱은 충분히 제 역할을 한 것이라고 봤습니다.

## 앞으로 개선하고 싶은 부분

오토 밸런싱은 한 번 만들고 끝나는 기능이라기보다, 실제 데이터를 보면서 계속 다듬어야 하는 기능에 가깝습니다. 앞으로는 이런 부분들을 개선해보고 싶습니다.

- 실제 경기 결과를 바탕으로 한 밸런스 정확도 보정
- 특정 유저의 포지션별 성과 반영
- 이전 게임에서 같은 팀이었던 조합을 피하는 기능
- 팀 평균 점수뿐 아니라 최고 점수 유저 편중도 고려
- 방장이 밸런스 기준을 선택할 수 있는 옵션

특히 내전 기록이 충분히 쌓이면, 단순 티어보다 더 좋은 기준을 만들 수 있을 것 같습니다. 예를 들어 NEXUS 내부 전적 기준으로 "내전 승률", "포지션별 성과", "최근 경기력" 같은 데이터를 활용할 수 있을 것입니다.

## 마무리

오토 밸런싱은 겉으로 보면 "버튼 누르면 팀 나눠주는 기능"입니다. 하지만 실제로 만들어보면 생각보다 많은 고민이 들어갑니다.

랜덤으로만 나누면 불공평하고, 점수만 맞추면 포지션이 망가지고, 가장 좋은 조합만 고르면 매번 비슷한 팀이 나옵니다. 그래서 NEXUS에서는 티어, LP, 선호 포지션을 참고하면서도 어느 정도 랜덤성을 함께 반영하는 방향으로 오토 밸런싱을 구현했습니다.

아직 완벽한 기능은 아니지만, 가볍게 내전을 열고 싶은 분들에게는 꽤 유용한 기능이 될 거라고 기대하고 있습니다.
